# JS 原型继承原理图解

![JS 原型继承原理图解](/img/js-prototype-inheritance-diagram.png)

JavaScript 的继承模型不是传统面向对象语言里的“类继承”起步，而是基于原型对象和原型链。

理解原型继承，核心要抓住三组关系：

- 实例对象通过 `__proto__` 指向构造函数的 `prototype`。
- 构造函数通过 `prototype` 保存实例共享的方法。
- 原型对象本身也有 `__proto__`，继续向上形成原型链。

当访问一个对象属性时，JavaScript 会先在对象自身查找，找不到再沿着原型链向上查找，直到 `Object.prototype`，最后到 `null`。

### 1. 构造函数、实例和 prototype

先看一个最基础的构造函数：

```js
function Person(name) {
  this.name = name;
}

Person.prototype.sayName = function () {
  return this.name;
};

const p = new Person("Tom");
```

这里有三类对象：

- `Person`：构造函数，本质也是函数对象。
- `Person.prototype`：原型对象，用来放实例共享方法。
- `p`：通过 `new Person()` 创建出来的实例对象。

它们之间的关系是：

```text
p.__proto__ === Person.prototype
Person.prototype.constructor === Person
```

因此 `p.sayName()` 能执行，是因为 `p` 自身没有 `sayName`，于是沿着 `p.__proto__` 找到了 `Person.prototype.sayName`。

### 2. new 操作符做了什么

`new Person("Tom")` 不是简单调用函数，它大致做了四件事：

```js
const obj = {};
obj.__proto__ = Person.prototype;
Person.call(obj, "Tom");
return obj;
```

更完整地说：

1. 创建一个空对象。
2. 把这个对象的原型指向构造函数的 `prototype`。
3. 用这个对象作为 `this` 执行构造函数。
4. 如果构造函数没有显式返回对象，就返回这个新对象。

这也是为什么实例能访问原型方法：`new` 在创建实例时已经把原型关系连好了。

### 3. 原型链查找规则

属性访问遵循一条稳定规则：

```text
对象自身
-> 对象.__proto__
-> 对象.__proto__.__proto__
-> ...
-> Object.prototype
-> null
```

例如：

```js
p.toString();
```

`p` 自身没有 `toString`，`Person.prototype` 上也没有，继续向上找到 `Object.prototype.toString`。

所以很多对象都能调用 `toString`、`hasOwnProperty`，不是因为它们自身都有这些方法，而是因为它们最终继承自 `Object.prototype`。

### 4. prototype 和 __proto__ 的区别

这两个概念经常混淆。

`prototype` 是函数对象上的属性：

```js
Person.prototype
```

它用于给通过这个构造函数创建的实例设置原型。

`__proto__` 是对象实例上的隐式原型指针：

```js
p.__proto__
```

它指向创建这个实例时使用的构造函数原型对象。

可以这样记：

```text
函数有 prototype，用来给实例当原型。
对象有 __proto__，用来指向自己的原型。
```

现代代码里不建议直接操作 `__proto__`，更推荐：

```js
Object.getPrototypeOf(p);
Object.setPrototypeOf(p, proto);
```

### 5. constructor 是什么

默认情况下，构造函数的原型对象会有一个 `constructor` 属性，指回构造函数本身：

```js
Person.prototype.constructor === Person;
```

如果直接重写 `prototype`，容易丢掉 `constructor`：

```js
Person.prototype = {
  sayName() {
    return this.name;
  },
};
```

此时 `Person.prototype.constructor` 不再默认指向 `Person`。如果依赖它，需要手动补回来：

```js
Person.prototype = {
  constructor: Person,
  sayName() {
    return this.name;
  },
};
```

实际业务中不建议过度依赖 `constructor` 判断类型，更常用 `instanceof`、品牌字段或更明确的数据结构。

### 6. instanceof 的本质

`instanceof` 判断的是：构造函数的 `prototype` 是否出现在实例对象的原型链上。

```js
p instanceof Person; // true
```

它等价于沿着原型链查找：

```text
p.__proto__
-> Person.prototype ?
```

只要链上能找到，就返回 `true`。

这也意味着，如果手动改了对象原型，`instanceof` 的结果可能变化。

### 7. 原型继承

原型继承的核心是让子类型原型对象指向父类型实例或父类型原型。

一个常见写法：

```js
function Animal(name) {
  this.name = name;
}

Animal.prototype.eat = function () {
  return `${this.name} eat`;
};

function Dog(name) {
  Animal.call(this, name);
}

Dog.prototype = Object.create(Animal.prototype);
Dog.prototype.constructor = Dog;

Dog.prototype.bark = function () {
  return `${this.name} bark`;
};

const dog = new Dog("Lucky");
```

这时原型链是：

```text
dog
-> Dog.prototype
-> Animal.prototype
-> Object.prototype
-> null
```

所以 `dog` 可以访问 `Dog.prototype.bark`，也可以访问 `Animal.prototype.eat`。

### 8. 为什么用 Object.create

不要直接这样写：

```js
Dog.prototype = Animal.prototype;
```

这样会让 `Dog.prototype` 和 `Animal.prototype` 指向同一个对象。给 `Dog.prototype` 添加方法时，会污染 `Animal.prototype`。

更好的方式：

```js
Dog.prototype = Object.create(Animal.prototype);
```

`Object.create` 会创建一个新对象，并把这个新对象的原型指向 `Animal.prototype`。这样既能继承父类方法，又不会直接修改父类原型对象。

### 9. class 的本质

ES6 的 `class` 语法让继承写法更接近传统面向对象：

```js
class Animal {
  constructor(name) {
    this.name = name;
  }

  eat() {
    return `${this.name} eat`;
  }
}

class Dog extends Animal {
  bark() {
    return `${this.name} bark`;
  }
}
```

但它的底层仍然是原型继承。

```text
dog
-> Dog.prototype
-> Animal.prototype
-> Object.prototype
-> null
```

`class` 只是更规范、更易读的语法，不是把 JavaScript 变成了传统 class-based 语言。

### 10. 静态方法和实例方法

实例方法放在 `prototype` 上：

```js
Person.prototype.sayName = function () {};
```

静态方法放在构造函数本身：

```js
Person.create = function () {};
```

对应到 class：

```js
class Person {
  sayName() {}

  static create() {}
}
```

区别是：

- 实例方法通过实例调用：`p.sayName()`。
- 静态方法通过类或构造函数调用：`Person.create()`。

### 11. 常见误区

#### 误区一：实例复制了 prototype 上的方法

实例并没有复制原型方法。它只是通过原型链查找到了方法。这样多个实例可以共享同一份函数，节省内存。

#### 误区二：__proto__ 和 prototype 是同一个东西

不是。

- `prototype` 属于函数。
- `__proto__` 属于对象实例。

它们有时指向同一个对象，但概念不同。

#### 误区三：class 没有原型链

有。`class` 只是语法糖，实例方法仍然在 `Class.prototype` 上。

#### 误区四：箭头函数适合做构造函数

箭头函数不能作为构造函数使用，没有自己的 `this`，也不能被 `new` 调用。

### 12. 实践建议

- 业务代码优先使用 `class` 或组合模式，避免手写复杂原型链。
- 理解原型链用于读框架源码、排查继承问题和理解语言机制。
- 不要直接修改内置对象原型，例如 `Array.prototype`、`Object.prototype`。
- 判断对象自有属性时使用 `Object.hasOwn` 或 `hasOwnProperty.call`。
- 创建无原型字典对象时可以使用 `Object.create(null)`。

### 总结

JavaScript 原型继承可以串成一条线：

```text
构造函数有 prototype
-> new 创建实例
-> 实例 __proto__ 指向构造函数 prototype
-> 访问属性时沿原型链向上查找
-> 子类型通过 Object.create 连接父类型 prototype
-> class / extends 本质仍然建立在原型链之上
```

掌握这条线之后，再看 `new`、`instanceof`、`class`、`extends`、`super` 和源码里的对象模型，就不会只是在背概念，而是在理解 JavaScript 的对象系统。
