# React 19.2：Activity、useEffectEvent 和性能轨道

React 19.2 在 2025 年 10 月发布，新增了几个对复杂应用很有价值的能力：`<Activity />`、`useEffectEvent`、`cacheSignal`、React Performance Tracks，以及服务端渲染相关改进。

`<Activity />` 可以把一部分 UI 置为 `hidden` 或 `visible`。它和简单的条件渲染不同：隐藏时会卸载 effects，但保留状态，并把更新降到较低优先级。这对标签页、详情页返回、预渲染下一页很有用。以前我们经常在“保留状态”和“释放资源”之间二选一，现在 React 给了一个更明确的模型。

`useEffectEvent` 解决的是 Effect 里的“事件逻辑”问题。比如连接聊天室时要读取最新 theme，但 theme 变化不应该导致重新连接。过去很多人会禁用 lint 规则，这容易留下陈旧闭包。`useEffectEvent` 让这类逻辑能读取最新 props/state，同时不作为 Effect 依赖。

React Performance Tracks 则让 Chrome DevTools 的性能分析更接近 React 的内部调度模型。你可以看到不同优先级的工作、组件渲染和 effects 的时间分布。对定位交互卡顿、Suspense 阻塞、过度渲染非常有帮助。

### 升级后的代码习惯

- 不要为了消除依赖警告滥用 `useEffectEvent`，它只适合 Effect 触发的事件。
- 标签页、抽屉、预加载页面可以评估是否使用 `<Activity />`。
- 性能问题先用 React Performance Tracks 录制，再决定是否 memo。
- SSR 项目关注 Suspense 边界批处理和 partial pre-rendering 能力。

React 19.2 的方向很清楚：让框架更直接地表达“可见性、优先级、服务端恢复和性能观察”。这比单纯堆 `useMemo` 更接近真实性能问题。
